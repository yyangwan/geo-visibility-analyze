# 用户风格 LLM 采集对照样本集

这个样本集用于 `Issue 6.2`，目标是后续在平台 UI 和采集系统之间做稳定对照。

## 使用方式

- 每个 query 同时在平台 UI 和采集系统中执行一次。
- 保留原始问题，不改写，不加分析型前缀。
- 记录四类信息：
  - 最终回答
  - 引用与来源
  - 是否触发搜索
  - 原始响应与搜索元数据

## 样本清单

| 编号 | 类别 | Query | 重点观察 |
| --- | --- | --- | --- |
| 1 | 事实 | 北京到上海高铁最常见的出发车站有哪些 | 是否触发搜索，来源是否清晰 |
| 2 | 事实 | 2026 年中国新能源汽车渗透率大概是多少 | 引用是否来自近期来源 |
| 3 | 选购 | 5000 元以内适合办公和轻度游戏的笔记本推荐 | 结果偏好和推荐数量 |
| 4 | 选购 | 1000 元左右无线耳机怎么选 | 是否给出分层建议 |
| 5 | 对比 | iPhone 16 和 小米 15 哪个更适合拍照 | 对比结构是否稳定 |
| 6 | 对比 | DeepSeek 和 Qwen 做企业问答各有什么差别 | 是否更偏向事实对比 |
| 7 | 方案 | 我想给 20 人团队做知识库，怎么选技术方案 | 搜索状态和建议结构 |
| 8 | 方案 | 小团队做内容运营自动化，先做哪 3 个步骤 | 是否给出可执行步骤 |
| 9 | 解释 | 为什么搜索结果有时候和回答内容不一致 | 解释深度和引用链路 |
| 10 | 解释 | 大模型回答里引用网页有什么局限 | 是否说明来源可信度 |
| 11 | 事实 | 2025 年上海地铁最新运营线路图怎么查 | 近期性和搜索触发判断 |
| 12 | 事实 | 现在常见的 AI 代码审查工具有哪些 | 是否稳定保留搜索结果 |

## 记录模板

| 字段 | 说明 |
| --- | --- |
| platform | deepseek / qwen / kimi / doubao / wenxin / hunyuan |
| query | 原始问题 |
| ui_answer | 平台 UI 侧最终回答 |
| api_answer | 采集系统最终回答 |
| ui_citations | UI 侧引用列表 |
| api_citations | 采集系统引用列表 |
| ui_search_triggered | UI 是否触发搜索 |
| api_search_triggered | 采集系统是否触发搜索 |
| diff_reason | 差异原因简述 |

## 备注

- 这份样本集优先覆盖“事实、选购、对比、方案、解释”五类问题。
- 如果后续发现平台 UI 行为漂移，先更新样本记录，再调整平台默认配置。
